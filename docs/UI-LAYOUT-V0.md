## 3. R2 → R3 纠错教训

| R2 写的 | R3 修正 |
|---|---|
| 5 角色草图全部带左侧 sidebar | sidebar 仅工作台/管理后台视图有;大盘视图无 |
| 顶栏未画"工作台"按钮 | 顶栏 [工作台] 按钮所有角色可见;[管理后台] sysadmin 独有 |
| `sidebarItemsForRole(role)` 单一函数 | 拆 `workspaceItemsForRole` + `ADMIN_ITEMS`;sidebar 内容由当前 path prefix 切换 |
| sysadmin 默认登录后只能看到 admin sidebar | sysadmin 也能进工作台(顶栏 [工作台] 按钮跳 `/console/dashboard`) |

**根本原因**:R2 把"工作台 sidebar"理解成全局 chrome,但 PRD §6.1 原文里"工作台 = 独立视图区域",sidebar 是它的局部组件。R2 ASCII 草图把这个误解视觉化了,直到 V0.3 代码跑通用户 review 才发现。

**协同拍下次的护栏**:画 ASCII 时先确认 layout 的"全局 chrome vs 局部组件"边界 — 顶栏是全局,sidebar 不一定。

## 6. V0.3 落地建议(R3 修正后)

V0.3 实现 layout 时建议节奏:

1. **Step 1** · `AppLayout` 顶栏 shell + 路由骨架(`<ProtectedRoute><AppLayout />` + `<RoleRedirect>` index)
2. **Step 2** · 顶栏 [工作台] / [管理后台] 按钮(sysadmin 才显示后者;[工作台] 跳 `workspaceDefaultForRole`)
3. **Step 3** · 条件 sidebar:
   - `isMap = path.startsWith('/map')` → 不渲染 `<Sider>`
   - `isWorkspace = path.startsWith('/console') || path.startsWith('/central')` → 渲染工作台 sidebar
   - `isAdmin = path.startsWith('/admin')` → 渲染管理后台 sidebar
4. **Step 4** · 5 角色首屏空壳 → 端到端跑通(沿用现有 stub 页)
5. **Step 5** · 双大盘地图画布(高德 SDK)
6. **Step 6** · 数据接入(回到 β/γ 业务实体)

**预估工作量**:Step 1-4 ≈ 1 天 · Step 5 ≈ 1 天 · Step 6 = β/γ 路径(2-3 天 each)

## 1 · chrome / view-local 边界声明

**边界判断源头**:`docs/PRD-user-led.md` §6.1 顶层信息架构(行号 1532-1583)+ §6.1 设计原则段(1577-1581)+ §6.3 页面地图(1601-1622)。

### 1.1 全局 chrome 清单

全局 chrome = 顶栏容器 + 全局 portal 三件套。但顶栏内部子控件**有显示条件区分**:大部分跨所有视图常显,「大盘切换」是 **上下文 chrome**(仅视图 ① 显示)。

#### 1.1.a 顶栏(`docs/PRD-user-led.md` §6.1:1535-1540)

| 子控件 | 位置 | 显示条件 | 行为 / 依据 |
|---|---|---|---|
| **Logo** | 左 | 跨所有视图常显 | **点击 = 永远进大地图(`/map/local`)** — 用户拍(2026-04-25 晚 ⚠️3,首次实施时误读为「§6.2 角色派发」,事后修正) |
| **大盘切换**(属地 ⇄ 政策) | 中 | **仅视图 ① 大盘显示**;离开 ① 后此控件隐藏 | 离开后想再切大盘:**点 Logo → `/map/local`(对所有角色一致)→ 控件回到中央** — 用户拍(⚠️3) |
| **工作台入口** | 右 | 跨所有视图常显 | §6.1:1538 |
| **个人中心**(头像下拉) | 右 | 跨所有视图常显 | §6.1:1539 |
| **管理后台入口** | 右(与「工作台入口」并列) | 仅 sys_admin 角色可见;跨所有视图常显 | §6.1:1540 + 用户拍(⚠️2)位置 = 顶栏右侧独立按钮 |

#### 1.1.b 全局 portal 三件套(用户拍 ⚠️1,归全局 chrome)

| 组件 | 触发条件 | 实现 |
|---|---|---|
| 全局 toast 通知 | 任意视图操作反馈(保存成功/失败/网络错) | AntD `message` portal |
| 全局 loading 蒙层 | 路由切换、长查询遮罩 | AntD `Spin` / 路由级 `Suspense` |
| 模态(Modal) | 二次确认、表单弹窗 | AntD `Modal` portal |

**关键边界**(本节核心结论):
- **顶栏**是全局 chrome 容器,但「大盘切换」是 **上下文 chrome 子控件** — 不是全局可见,只在视图 ① 显示
- **Logo** 不只是品牌标识,**承担「永远进大地图」入口职能**(对所有角色一致跳 `/map/local`);§6.2 角色派发**只用于登录后首次进入**(`/` 重定向 `RoleHomeRedirect`),不是 Logo 行为
- 全局 portal 三件套跨视图触发,放全局 chrome 层管理

## 2 · R1 全局骨架 ASCII

R1 = 全局 chrome 视角(view-agnostic),不画任何 view-local 细节。
因为「大盘切换」是 view-context-aware 子控件,R1 必须画 **三态** 才能完整表达 chrome 行为:
- **R1-α** 标准态(active = ② / ③ / ④,顶栏中央无大盘切换)
- **R1-β** 大盘特殊态(active = ①,顶栏中央显示大盘切换)
- **R1-γ** 登录态(active = ⑤,顶栏不显示)

视图占位区只画「[view-local 区域]」纯标识,**所有 view-local 细节(sidebar / 抽屉 / 左面板 / ➕📌)归阶段 C 的 R2 各视图 ASCII**。

> 双轨 PRD 实验的最终产出。基于 11 章用户版定稿 + AI 版独立产出 768 行 + D1-D8 速览笔记,做正式整合判断。

> **生成时间**:2026-04-23 夜

## 1. 摘要

**双轨实验核心结论**:

用户版与 AI 版在功能模块层面有较高重合(地图 / 政策 / 机会 / 任务 / 角色),但在**设计哲学层面存在 5 处根本性偏离**;**3 处角色定位重大分歧**;**1 套核心创新**(工具体系)AI 完全没想到。

**最关键的元学习发现**:让 AI 在没有用户业务上下文的情况下做政企工具 PRD,会自动往"**政企 CRM + 热力图皮**"的方向靠 —— 这是市面上 PRD 模板的主流路径(Opportunity / Engagement / Campaign / SupportRequest / Policy 建档 / GovContact)。

用户版的"**反 CRM 设计哲学**"(订阅 / 工具体系 / 涂层 / 决策层不上系统 / 不做地域硬隔离 / PMO 一双手)是**主动选择**而非无意识 —— 这是双轨实验最大的价值。

## 2. 8 维度对比矩阵概览

| 维度 | 用户版倾向 | AI 版倾向 | 分歧严重度 |
|---|---|---|---|
| **D1 覆盖范围** | 工具体系 + 涂层 + 5 点语义 + 周观测 + ExportRecord 水印 | Opportunity + Engagement + Campaign + SupportRequest + Policy 建档 + GovOrg/Contact | 高 |
| **D2 角色** | PMO=一双手 / 不做属地隔离 / 决策层非系统用户 | PMO=独立中枢 / 属地按省市硬隔离 / 决策层是用户(有移动端高管视图) | 极高(3 处重大) |
| **D3 优先级** | 单维度热力 + 移动端 H5 P0 + 不做高管视图 | 多维度热力 P0 + 移动端 P1 + 高管视图 P0 + 月报 P1 | 中高 |
| **D4 字段命名** | Pin / Visit / PolicyTheme / Tool / PlanPoint | Opportunity / Engagement / Policy / 无工具 / Task | 命名重合度 ≈ 0 |
| **D5 状态机** | Pin 3 态 / PolicyTheme 3 态 / 蓝点转色机制 | Opportunity 6 态 / Policy 5 态 / Task 5 态 / SupportRequest 4 态 + 驳回 | AI 版偏 CRM 业务生命周期,用户版偏地图元素状态 |
| **D6 边界** | 设计哲学层面("数据录入≠系统"等三大不做) | 功能模块层面(BI / 财务 / 海外 / 政务对接) | 互补可合并 |
| **D7 非功能** | 偷懒 MVP 默认值 + 软触发回调 | 99.9% SLA + 字段级权限 + 弱网 P1 + 批量导入 | AI 版具体度高,用户版后置 |
| **D8 盲点互补** | AI 补:Campaign / GovContact / 异动告警 / 字段级权限 / 月报 | 用户补:工具体系 / 涂层 / 蓝点转色 / 决策层非系统 / 政策订阅模式 | 双向有学习价值 |

## 3. 三大角色重大分歧 → 整合建议

| 角色议题 | 用户版 | AI 版 | 整合建议 | 理由 |
|---|---|---|---|---|
| **PMO 定位** | 负责人的一双手,操作执行,常一人兼任 | 独立项目管理 + 战略中枢 | **保留用户版** | 用户对自己组织实情有第一手认知;AI 是市场模板默认值 |
| **属地 GA 数据域** | 不做地域硬隔离,人是主责实体,业务划分交叉 | 按省/市硬隔离,跨域需临时授权 | **保留用户版** | 用户 2026-04-23 已在 1.3 权限基线明确拍板;实际组织架构灵活,硬隔离会妨碍协作 |
| **决策层是否系统用户** | 非系统用户,线下汇报 + 导出截图 | 系统用户,有移动端高管视图 | **保留用户版** | AI 版自评判断点 #4 也承认此处可能错判("现实中高管常用的是 PMO 每周塞进邮箱的 PPT,自主使用率可能极低") |

## 4. 五处用户版"反 CRM"设计哲学(全部保留)

| # | 用户版选择 | AI 版默认走向 | 用户版决策时点 |
|---|---|---|---|
| 1 | **政策订阅**(主题 + 外部覆盖清单) | Policy 一等实体建档 + 解读状态机 | 2026-04-23 场景 2 拍板 |
| 2 | **中台响应在系统外**(IM / 电话 / 邮件) | SupportRequest 队列 + 4 状态 + 驳回 | 用户版 1.4 明确写出 |
| 3 | **决策层非系统用户** | 决策层是用户 + 移动端高管视图 | 0.1 / 1.1 / 1.5 |
| 4 | **工具体系核心差异化**(成品文档 + 调用接口 + 级联匹配) | 完全无此概念 | 1.4(b) / 4.5 / D 模块 |
| 5 | **地图元素一等交互**(图钉 / 蓝点 / 拜访点 + 涂层) | 地图是 CRM 可视化外皮 | 整体设计哲学 |

## 5. AI 版补盲(可借鉴清单 + 优先级建议)

| AI 版补出的能力 | 用户版当前状态 | 借鉴建议 | 优先级 |
|---|---|---|---|
| **Campaign 跨区域战役建模** | 用户版 1.3 已拍"跨地域协作是常态",但未对协作项目本身建模 | **建议进 V1.0 后评估**,作为 F13 候选(暂定);若 V1.0 真实出现跨省高频协作,V1.1 落地 | F13 候选 |
| **月度报告一键生成** | 用户版 F7 周报远期(V1.1) | **建议把月报与 F7 同期或更前**——PMO 1.2 职责"协助导出材料给决策层"直接对应;比 AI 周报更优先 | F7 增强或新开 F14 |
| **GovOrg / GovContact 关键人档案** | 用户版仅 Visit.contact_person 字段级 | **暂不采纳**——用户版场景 1-5 都没"查对接人档案"的动作;若 V1.0 后业务确认有"关系经营"诉求,再加 | 后置评估 |
| **字段级敏感保护** | 用户版仅导出级水印 | **暂不采纳**——前提是关键人档案先建立,与上一条联动 | 后置评估(联动) |
| **异动告警**(热力大幅变化) | 用户版 F2 监管图层有类似但仅限监管事件 | **建议作为 F2 增强**——监管图层落地时扩展为"广义异动" | F2 增强 |
| **时间轴回放** | 用户版无 | **不采纳**——视觉强但业务价值不明确 | 不采纳 |
| **关系图谱可视化** | 用户版无 | **不采纳**——同上 | 不采纳 |

### 7.1 AI 版让你最意外的揣测

**主对话推荐选**:**决策层当系统用户 + 给高管视图**

理由:这条最颠覆用户版,且 AI 自己也在判断点 #4 承认可能错判;能最直接说明"AI 在没有用户行为洞察时,默认假设最理想用户使用模式"。

- 决策层当系统用户(主对话推荐)

### 7.3 哪几类决策绝对不能让 AI 自由发挥

**主对话推荐通用规律**:涉及以下三类的决策,AI 倾向往市场模板默认值靠,必须用户拍板:

1. **角色职责定位**(如 PMO / 决策层)—— AI 不知道你的组织实情
2. **权限边界**(如数据域硬隔离 / 字段级权限)—— AI 默认走最严谨的传统 CRM 模式
3. **核心业务哲学**(如政策建档 vs 订阅、工具体系 vs 文档附件)—— AI 不知道你的业务真实工作流

### 7.4 哪几类决策可以 AI 自由发挥

**主对话推荐通用规律**:涉及以下三类的决策,AI 输出质量较高:

1. **字段级细节 / 状态机状态数**(已有最佳实践)
2. **非功能项数值默认值**(如 SLA / 性能阈值)
3. **UI 文案 / 交互细节**(行业通用模式)
